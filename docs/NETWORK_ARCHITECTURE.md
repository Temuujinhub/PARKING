# Сүлжээний архитектур — камер холболт (agent-гүй шийдэл)

## Асуудлын оношилгоо (2026-07-07)

| Чиглэл | Үр дүн |
|---|---|
| VPN laptop → сервер 172.16.100.21 | ✅ |
| VPN laptop → камер 10.0.113.10 | ✅ |
| Сервер → камер | ✅ |
| **Камер → дотоод сервер 172.16.100.21** | ❌ ХААЛТТАЙ |
| **Камер → интернэт (test.easy-parking.mn)** | ✅ |

**Үндсэн шалтгаан:** Камерын VLAN (10.0.113.0/26) нь интернэттэй ч дотоод серверийн
VLAN (172.16.100.0/24)-ээс тусгаарлагдсан. Байгууллагын FortiGate дээр камер→сервер
урсгал зөвшөөрөгдөөгүй (аюулгүй байдлын зориудын сегментчлэл).

## SSH-ээр засаж болох уу? — ҮГҮЙ

Сервер аль хэдийн бүх урсгал хүлээн авдаг (ufw inactive, ping ажилладаг). Камерын пакет
серверт огт хүрэхгүй байгаа нь VLAN хоорондын FortiGate policy. Серверийн SSH тохиргоо
үүнийг өөрчилж чадахгүй — зөвхөн байгууллагын сүлжээний админ.

## Windows agent хэрэгтэй юу? — Заавал биш

Easy-park agent-ыг яг энэ VLAN тусгаарлалтыг гүүрдэхэд ашигладаг. Бидний шууд ITSAPI
холболт нь илүү цэвэр — камер серверт хүрч чаддаг л бол. Тиймээс:

### Шийдэл A — Нэг firewall дүрэм (ХАМГИЙН САЙН)
Байгууллагын IT-д:
> Камерын сегмент 10.0.113.0/26 → сервер 172.16.100.21 TCP/80 гарах урсгал зөвшөөрөх.

Камер→сервер (ANPR) нээгдэнэ, сервер→камер (хаалт) аль хэдийн ажилладаг.
**Agent-гүй, cloud-гүй, бүх зүйл on-prem.**

### Шийдэл B — Public/cloud сервер
Камер интернэтээр public серверт ANPR илгээнэ. Гарах хаалтны команд (cloud→камер)
NAT-ын ард хүрэхгүй тул орох талд камерын relay, гарах талд нэмэлт шийдэл. 40 зогсоолд
масштаблахад тохиромжтой.

### Шийдэл C — Хөнгөн agent/edge relay
Камерын сүлжээн дэх жижиг Linux төхөөрөмж. Зөвхөн сүлжээ өөрчлөх боломжгүй үед.

## Хаалтны хоёр чиглэлийн шаардлага

| Үйлдэл | Чиглэл | Дотоод сервер | Cloud сервер |
|---|---|---|---|
| ANPR дата | камер→сервер | Firewall дүрэм хэрэгтэй | ✅ интернэтээр |
| Хаалт нээх (CGI) | сервер→камер | ✅ ажилладаг | ❌ NAT-ын ард |

→ Дотоод сервер + Шийдэл A нь хоёр чиглэлийг бүрэн шийднэ. Cloud нь ANPR-д сайн ч
хаалтанд local relay/агент шаардана.

## Зөвлөмж

- **Энэ зогсоол:** Шийдэл A (firewall дүрэм). Дотоод сервер хэвээр, agent хэрэггүй.
- **40 зогсоол:** сүлжээ бүрийн бүтцээр — эсвэл зогсоол бүрд firewall дүрэм, эсвэл
  cloud + гарах хаалтанд relay. Тус бүрээр төлөвлөнө.
- **Хүлээх зуур:** камерыг test.easy-parking.mn руу чиглүүлж ANPR pipeline-ыг батлах
  (камер интернэттэй тул ажиллана).
