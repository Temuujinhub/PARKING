# Тоног төхөөрөмж холбох заавар (Dahua ANPR Kit + DZBL-A)

Танай талбай дээрх төхөөрөмж: **Dahua Access ANPR Kit DHI-IPMECS-2234-IZ**
(камер + LED дэлгэц + DZBL-A barrier контроллер нэг багц).

## 1. Сүлжээний бэлтгэл

- Камер, контроллер, сервер нэг сүлжээнд (эсвэл VPN) байх ёстой
- Камерт статик IP өгнө (жишээ: 192.168.1.108)
- Серверийн хаяг: `152.42.235.199` (интернэтээр) — камер интернэт гаралттай байх шаардлагатай,
  эсвэл талбай дээрх edge PC-ээр дамжуулна

## 2. Камерын ITSAPI callback тохиргоо

1. Системд нэвтэрч **Тохиргоо → Төхөөрөмж** хуудаснаас камер бүрийн `device_key`-г хуулна
   (жишээ: `cam-entry-site01`)
2. Камерын Web UI (браузераар IP руу нь орно):
   - **Setting → Network → Platform Access → ITSAPI** (зарим firmware дээр *Platform Server*)
   - Enable ✓
   - Server: `test.easy-parking.mn` (эсвэл `152.42.235.199`), Port: `80` — HTTP-ээр зөвшөөрөгдсөн
   - Path: `/api/lpr/callback?device_key=cam-entry-site01`  ← орох камерын түлхүүр
3. Гарах камерт мөн адил, гарах камерын device_key-ээр
4. **Setting → ANPR**: Plate detection zone, trigger line-ийг эгнээндээ тааруулна
5. Confidence threshold: 90+
6. NTP цагийн синк идэвхжүүлнэ (Setting → System → Date&Time)

## 3. Barrier холболт — 2 арга

### Арга А: Камер шууд relay-ээр нээх (одоогийн kit-ийн стандарт холболт)

Камерын ALM_OUT1 → DZBL-A OPEN input (та зургаар ирүүлсэн монтаж яг ийм байна):

```
ITC/IPMECS камер            DZBL-A контроллер
ALM_OUT1  ────────────────►  OPEN input
COM1      ────────────────►  COM/GND
Pulse: 1000–2000ms (Setting → ANPR → Barrier Control)
```

Энэ горимд whitelist (гэрээт жолооч)-ийг камерын өөрийн whitelist руу нэмж болно —
интернэт тасарсан үед ч ажиллана.

### Арга Б: Сервер CGI командаар нээх (системээр бүрэн удирдах — санал болгож буй)

1. **Тохиргоо → Төхөөрөмж** дээр barrier төхөөрөмжийн IP-г оруулна
2. Сервер `.env` дээр:
   ```
   PARKING_BARRIER_MOCK=false
   PARKING_BARRIER_USERNAME=admin
   PARKING_BARRIER_PASSWORD={контроллерийн нууц үг}
   ```
3. `systemctl restart parking-backend`
4. Систем дараах командыг ашиглана:
   ```
   GET http://{IP}/cgi-bin/accessControl.cgi?action=openDoor&channel=1&UserID=101&Type=Remote
   (Digest authentication)
   ```

**Зөвлөмж:** Орох тал — Арга А (хурдан, серверээс хамааралгүй),
Гарах тал — Арга Б (төлбөр баталгаажсаны дараа л нээгдэнэ).
Гарах камерын relay-г DZBL-A-тай БҮҮ холбо, эсвэл камерын barrier control-ийг OFF болгоно.

## 4. Туршилтын дараалал

1. `Тохиргоо → Төхөөрөмж` дээр камерын IP зөв эсэх
2. Туршилтын машин оруулж:
   - Системийн **Лог → Камерын event лог** дээр event ирсэн эсэх
   - Хяналтын самбарт ENTRY event шууд гарч ирэх эсэх
3. Orох barrier нээгдсэн эсэх (Арга А бол камераас, Б бол системээс)
4. Гарах туршилт: гарах камерт уншуулж **Касс** дэлгэцэд машин гарч ирэхийг шалгана
5. Төлбөр (бэлэн/QPay) → гарах barrier нээгдэх эсэх
6. `Хаалтны удирдлага → Командын түүх` дээр SUCCESS бүртгэгдсэн эсэх

## 5. QR стенд

**Тохиргоо → Зогсоол → QR** товчоор зогсоол бүрийн QR-ийг гаргаж хэвлэнэ.
URL: `http://152.42.235.199/pay?site=SITE01`
Гарах хаалтны өмнө, жолооч машинаасаа харах өндөрт (100–120см) байрлуулна.
Шөнө гэрэлтүүлэгтэй байх шаардлагатай.
